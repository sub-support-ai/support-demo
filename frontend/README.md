# Точка поддержки Frontend

MVP UI для сценария:

1. вход или регистрация;
2. чат с AI;
3. эскалация в тикет при `requires_escalation=true`;
4. подтверждение pre-filled тикета;
5. список тикетов пользователя.

## Валидация регистрации

- email проверяется на формат на фронте и через `EmailStr` на backend;
- повторная регистрация уже существующего email возвращает ошибку API и
  подсвечивает поле email;
- логин: 3-32 символа, латиница, цифры, точка, дефис или подчёркивание;
- пароль: 8-128 символов, минимум строчная, заглавная, цифра и спецсимвол,
  без пробелов.

Проверка реального существования почтового ящика требует отдельного flow с
кодом подтверждения по email; в MVP это не реализовано.

## Запуск

```powershell
copy .env.example .env.local
npm install
npm run dev
```

По умолчанию API ожидается на:

```env
VITE_API_BASE_URL=http://localhost:8000/api/v1
```

Перед запуском фронтенда поднимите `RestAPI` и задайте в backend `.env`:

```env
CORS_ORIGINS=http://localhost:5173
AI_SERVICE_URL=http://localhost:8001
POSTGRES_HOST=localhost
```
